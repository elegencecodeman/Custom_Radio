# 情绪与音乐规则

> 用途：给个人助理 / 音乐 Agent / Vibe Coding 项目使用。  
> 目标：根据用户的情绪、场景、时间段和语言偏好，选择更符合个人听歌口味的音乐，而不是泛泛推荐热门歌。

---

## 1. 用户音乐人格总则

用户的听歌核心不是“单一风格偏好”，而是以 **情绪叙事 + 华语黄金年代 + 港乐情怀 + R&B/Soul 基因** 为主轴。

### 核心画像

- 主要偏好：粤语流行、台湾华语流行、华语 R&B / Soul、欧美流行、J-Pop / Anime OST。
- 主要语言：粤语、普通话、英语、日语。
- 主要情绪：情歌、感伤、怀旧、深情、治愈。
- 主要时代偏好：
  - 1990s 经典港乐 / 华语经典
  - 2000s 台湾华语黄金时代
  - 2010s 主流华语与欧美流行
  - 2020s 港乐新生代与 J-Pop
- 核心艺术家锚点：
  - 陈奕迅、周杰伦、陶喆、林俊杰、王力宏
  - MC 张天赋、方大同、卫兰、容祖儿、周柏豪
  - Taylor Swift、Bruno Mars、YOASOBI、Aimer、五月天

---

## 2. 推荐优先级

当系统需要推荐音乐时，按以下优先级判断：

```text
明确情绪 > 当前场景 > 时间段 > 能量需求 > 语言偏好 > 艺术家偏好 > 新鲜度
```

### 判断规则

1. 如果用户明确说出情绪，例如“emo”“烦”“失落”“想听点燃的”，优先根据情绪选歌。
2. 如果用户明确说出场景，例如“写代码”“开车”“睡前”“直播前”，优先根据场景选歌。
3. 如果没有明确情绪，结合时间段推断：
   - 深夜：港乐、情歌、治愈、低能量
   - 上午：清爽、轻快、不要太 emo
   - 下午：R&B、轻节奏、适合专注
   - 晚上：怀旧、故事感、情绪沉浸
4. 如果无法判断，默认推荐：
   - 华语 / 粤语情歌
   - R&B / Soul
   - 有歌词叙事、有旋律记忆点的歌曲
5. 不要优先推荐纯电子舞曲、随机 K-Pop、重金属、无歌词氛围音乐，除非用户明确要求。

## 4. 场景规则

### 4.1 深夜独处

默认推荐：
- 港乐深夜场
- 陈奕迅、MC 张天赋、卫兰、周柏豪
- 慢歌、感伤、歌词强

推荐格式：
```text
你现在适合听：粤语深夜情歌 + 低能量治愈
先放陈奕迅，再接一两首新港乐，最后用温柔女声收尾。
```

---

### 4.2 开车 / 通勤

默认推荐：
- 五月天、周杰伦、Taylor Swift、Bruno Mars
- 中高能量
- 旋律抓耳
- 不要太丧

推荐格式：
```text
通勤适合：华语青春感 + 欧美流行提神
控制情绪浓度，不要连续放失恋歌。
```

---

### 4.3 睡前

默认推荐：
- 林俊杰、王力宏、李荣浩、梁静茹、Aimer
- 低能量、温柔、少鼓点
- 不推荐强摇滚和快歌

推荐格式：
```text
睡前适合：温柔华语 + 轻 R&B + 少量空灵日系
目标是降噪，不是把情绪推得更重。
```

---

### 4.4 运动 / 提神

默认推荐：
- 五月天、BEYOND、Queen、Bruno Mars、G.E.M.
- 中高能量
- 明亮、积极、有节奏

推荐格式：
```text
运动适合：摇滚青春感 + 欧美 Funk/Pop
用节奏把状态拉起来，不用太苦情。
```

---

### 4.5 直播 / 上播前

默认推荐：
- Bruno Mars、Taylor Swift、五月天、周杰伦轻快曲
- 要提升表达欲和精神状态
- 不要太悲伤、不要太催眠

推荐格式：
```text
上播前适合：轻快流行 + 提神节奏
目标是让状态打开，避免深夜情绪歌。
```

---

### 4.6 复盘 / 思考 / 写文案

默认推荐：
- 陶喆、方大同、林俊杰、李荣浩
- R&B / Soul
- 稳定节奏
- 情绪有质感但不抢注意力

推荐格式：
```text
复盘适合：华语 R&B + 都市情绪
既保留审美，又不破坏思考连续性。
```

## 5. 时间段规则

### 上午

- 推荐：轻快、清爽、不要太 emo。
- 优先：周杰伦、陶喆、孙燕姿、Taylor Swift。
- 避免：陈奕迅深夜苦情连续播放。

### 下午

- 推荐：专注、节奏稳定、R&B / Pop。
- 优先：陶喆、方大同、王力宏、Charlie Puth。
- 避免：太慢、太伤、太炸。

### 晚上

- 推荐：怀旧、情歌、叙事感。
- 优先：周杰伦、林俊杰、五月天、梁静茹、孙燕姿。
- 可逐步转入港乐。

### 深夜

- 推荐：粤语情歌、治愈、低能量。
- 优先：陈奕迅、MC 张天赋、卫兰、张学友、李荣浩。
- 避免：强 EDM、过燃摇滚、过度欢快欧美流行。

## 6. 语言比例规则

在没有明确要求语言时，推荐歌单的语言比例可按以下方式控制：

```text
华语 / 粤语：60% - 70%
英语：20% - 30%
日语：5% - 10%
韩语：0% - 3%
```

### 语言选择逻辑

1. 情绪复杂时，优先粤语 / 普通话，因为用户更重视歌词叙事。
2. 需要提神时，可增加英语 Pop / Funk。
3. 需要幻想感、动漫感、夜跑感时，可增加日语 J-Pop。
4. 不要主动大量推荐韩语歌，除非用户明确要求。

## 9. 反向规则：不要这样推荐

### 不要把用户简单归类为“只听伤感情歌”

用户确实偏好情歌、感伤和叙事歌词，但同时也有：
- 欧美流行
- R&B / Soul
- J-Pop / Anime OST
- 五月天 / BEYOND / Queen 等摇滚与励志内容

因此推荐时要避免长期单一 emo 化。

### 不要忽视 R&B / Soul

陶喆、方大同、王力宏、Bruno Mars 是用户品味中的关键结构。  
当用户说“舒服”“高级”“松弛”“写代码”“工作”时，R&B / Soul 通常比普通情歌更合适。

### 不要用泛热门替代个人偏好

错误示例：
```text
最近很火，所以推荐。
```

正确示例：
```text
这首歌符合你的粤语情绪叙事偏好，同时能接上陈奕迅 / MC 张天赋这一类港乐线索。
```

### 不要大量推荐纯 EDM

用户有少量电子 / EDM 偏好，但不是主轴。  
除非用户明确说“要蹦迪”“要电音”“要很炸”，否则 EDM 只能作为补充。

## 11. Agent 判断伪代码

```pseudo

IF user_mood is explicit:
    route_by_mood(user_mood)
ELSE IF scene is explicit:
    route_by_scene(scene)
ELSE:
    infer_from_time(time_of_day)

IF language_request exists:
    apply_language_filter(language_request)
ELSE:
    keep_language_ratio(CN_CANTONESE=0.65, EN=0.25, JP=0.08, KR=0.02)

IF energy_request exists:
    adjust_energy(energy_request)
```
